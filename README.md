Prosjektuke 1
Hva jeg skal lage
Jeg vil lage en personlig nettside/portefølje der jeg viser hvem jeg er og prosjekter jeg har laget.
Prototype / idé
En enkel nettside med navn, bilde, prosjekter og kontaktinfo.
Teknologier
•	HTML og CSS
•	Visual Studio Code
•	GitHub
Kompetanse i fagene
•	Drift: publisere siden på GitHub
•	Utvikling: kode nettsiden
•	Brukerstøtte: teste og forbedre basert på tilbakemeldinger
Plan
1.	Lage enkel skisse (i Figma eller på papir)
2.	Lage HTML-struktur
3.	Style med CSS
4.	Laste opp til GitHub
Logg – prosjektoppgave (uke 50–51)

Uke 50:
I uke 50 startet jeg ordentlig med prosjektoppgaven. Først brukte jeg tid på å planlegge hva prosjektet skulle inneholde og hvordan jeg skulle bygge det opp. Jeg jobbet med filstruktur og gjorde klart prosjektet på PC-en min. Jeg testet også litt lokalt for å se at ting fungerte som de skulle. Underveis lærte jeg mer om hvordan prosjektet henger sammen, og hva som må være på plass før man kan hoste det på en server.

Senere i uka begynte jeg å forberede Raspberry Pi. Jeg sjekket oppsettet, koblet meg til via terminal og passet på at jeg hadde tilgang. Det var litt nytt for meg, så jeg måtte bruke tid på å forstå hvordan Pi-en fungerer som en liten server. Jeg fikk noen feil, men jeg skrev dem ned og prøvde å finne løsninger.

Uke 51 (mandag–onsdag):
Denne uka fortsatte jeg med prosjektet og fokuserte mer på å få det klart for hosting. Jeg ryddet i filer og sjekket at alt jeg trengte faktisk var med. Jeg testet prosjektet flere ganger på PC før jeg flyttet det over, så jeg var sikker på at det ikke var feil derfra.

Jeg jobbet også med feilsøking og lærte mer om hvordan tjenester kjører i bakgrunnen. Noen ganger fungerte ikke ting med en gang, og da måtte jeg prøve forskjellige løsninger. Det tok tid, men jeg følte at jeg lærte mye av det.

Uke 51 (torsdag – i dag):
I dag fikk jeg hostet prosjektet mitt over på Raspberry Pi. Det var hovedmålet mitt, og jeg er fornøyd med at jeg klarte det. Først overførte jeg prosjektfilene, og deretter testet jeg om prosjektet faktisk kjørte på Pi-en. Det fungerte ikke helt med en gang, så jeg måtte feilsøke og sjekke om alt var riktig satt opp.

Til slutt fikk jeg prosjektet til å fungere, og det føltes bra. Jeg lærte mye om hosting, servere og hvordan man jobber mer praktisk med IT. Jeg har også blitt bedre til å være tålmodig og prøve å løse problemer selv. Neste steg blir å forbedre prosjektet videre og gjøre det mer ferdig
